Web Cache Poisoning Research Index

            

                

                    
🔴 Unkeyed Header Reflection

                    
Testing if application metadata (e.g., X-Forwarded-Host) is reflected in the
                        response but ignored by the cache key, allowing for resource-path hijacking.

                

                

                    
🔴 Cache-Stored XSS

                    
Injecting JavaScript into unkeyed headers to poison the cache with a malicious response that
                        executes script in the browsers of all subsequent visitors.

                

                

                    
🟡 Internal Cache Deception

                    
Tricking the cache into storing sensitive responses from absolute paths by manipulating how the
                        internal framework resolves relative file identifiers.

                

                

                    
🟡 Parameter Cloaking Logic

                    
Exploiting cases where a cache ignores certain query parameters (e.g., utm_) while
                        the backend server uses them to trigger specific logic branches or outputs.
