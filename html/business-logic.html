Business Logic Research Index

            

                

                    
🔴 Excessive Trust in Data

                    
Testing if an application blindly accepts sensitive data from the client, such as product prices
                        or discount codes in a POST request, without server-side verification.

                

                

                    
🔴 Multi-Step Logic Bypass

                    
Testing if mandatory steps in a process (e.g., verifying a code) can be bypassed by directly
                        accessing the subsequent URL in the sequence.

                

                

                    
🟡 Logic Parameter Tuning

                    
Submitting unexpected values (e.g., negative quantities, extremely large integers) to see if they
                        cause unintended side effects in the business logic flow.

                

                

                    
🟡 Concurrency (Race conditions)

                    
Triggering parallel requests to a single action node to exploit gaps between a "check" (e.g.,
                        have balance?) and an "action" (e.g., withdraw funds).
