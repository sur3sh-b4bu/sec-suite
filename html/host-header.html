Host Header Research Index

            

                

                    
🔴 Password Reset Poisoning

                    
Manipulating the Host header during a reset request to force the application to
                        generate links pointing to an attacker-controlled domain.

                

                

                    
🔴 SSRF via Host Header

                    
Exploiting cases where the application or proxy uses the Host header to route
                        requests to internal servers or localhost services.

                

                

                    
🟡 Duplicate Header Collision

                    
Sending two Host headers in one request to exploit inconsistencies in how different
                        middleware layers prioritize the values.

                

                

                    
🟡 Absolute Request Routing

                    
Using an absolute URL in the request line (e.g., GET http://victim.com/) which might
                        take precedence over the Host header value.
