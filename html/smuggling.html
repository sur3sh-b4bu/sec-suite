Request Smuggling Research Index

            

                

                    
🔴 CL.TE Desync

                    
Exploiting cases where the front-end server uses Content-Length but the back-end
                        uses Transfer-Encoding, allowing a portion of the request to be 'smuggled' to the
                        next user's transaction.

                

                

                    
🔴 HTTP/2 Downgrade

                    
Exploiting the translation between HTTP/2 (front-end) and HTTP/1.1 (back-end). Smuggling occurs
                        when the front-end fails to sanitize content-length or
                        transfer-encoding headers in H2 streams.

                

                

                    
🟡 TE.CL Desync

                    
The inverse attack where the front-end uses Transfer-Encoding and the back-end uses
                        Content-Length, causing the back-end to pause and wait for the remaining bytes of a
                        smuggled request.

                

                

                    
🟡 Web Cache Poisoning

                    
Using a smuggled request to force the back-end to return a malicious response that the front-end
                        cache then incorrectly associates with a legitimate URL.
