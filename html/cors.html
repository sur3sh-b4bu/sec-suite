CORS Misconfiguration Index

            

                

                    
🔴 Origin Reflection

                    
The server reflects any domain in the Origin header into the
                        Access-Control-Allow-Origin response, creating a universal trust flaw.

                

                

                    
🔴 Null Origin Vulnerability

                    
Trusting the null origin allows exploits from sandboxed iframes or local files to
                        exfiltrate data.

                

                

                    
🟡 Regex Trust Flaws

                    
Poorly implemented regex (e.g. victim.com) allowing bypass via
                        attacker-victim.com or victim.com.evil.net.

                

                

                    
🟡 Insecure Protocol Bridge

                    
Allowing http://domain.com to access resources on https://domain.com,
                        exposing credentials to MITM.

                

            


            
            
Defense Heuristics

            

                

                    
✓ Whitelist Identity

                    
Explicitly check the Origin against a hard-coded whitelist of trusted domains. Never
                        use reflection.

                

                

                    
✓ Deny Credentials

                    
Disable Access-Control-Allow-Credentials unless absolutely required for cross-origin
                        state management.

                

                

                    
✓ Strict CORS Filter

                    
Implement server-side middleware that uniformly rejects any non-whitelisted cross-origin request
                        before processing.
