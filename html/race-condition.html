Race Condition Research Index

            

                

                    
🔴 Limit Overrun

                    
Executing multiple transactions (e.g., gift card redemptions) simultaneously before the database
                        can update the balance, allowing for more usage than permitted.

                

                

                    
🔴 Single-Packet HTTP/2

                    
Using HTTP/2 multiplexing to send dozens of requests within a single TCP packet, ensuring they
                        reach the server at exactly the same time, bypassing network jitter.

                

                

                    
🟡 TOCTOU Windows

                    
Exploiting the gap between when a server checks a condition (e.g., file existence) and when it
                        uses it, by changing the state in that specific micro-window.

                

                

                    
🟡 Partial Object Desync

                    
Racing during multi-step object initialization where values might be partially written but
                        accessible by other concurrent threads (e.g., in Node.js or Java).
