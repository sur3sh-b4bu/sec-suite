WebSocket Research Index

            

                

                    
🔴 CSWSH Hijacking

                    
Cross-Site WebSocket Hijacking occurs when an application relies solely on cookies for
                        authentication and doesn't validate the Origin header, allowing an attacker to
                        initiate a socket from their own site.

                

                

                    
🔴 Socket Manipulation

                    
Testing for logic vulnerabilities by intercepting and modifying WebSocket messages. This includes
                        spoofing identity (user_id) or modifying transaction values (amount) in flight.

                

                

                    
🟡 XSS via Stream

                    
Injecting HTML/JavaScript payloads into WebSocket messages that are subsequently rendered by
                        another user's browser, bypassing traditional HTTP output filters.

                

                

                    
🟡 Origin Bypass logic

                    
Testing if the server can be tricked into accepting unauthorized origins through techniques like
                        null origin, subdomains, or regex-based misconfigurations.
