JWT Research Index

            

                

                    
🔴 Algorithm None Attack

                    
Modifying the header to use alg: none. Some insecure implementations skip signature
                        verification entirely if this algorithm is specified.

                

                

                    
🔴 RSA/HMAC Confusion

                    
Changing the algorithm from RS256 to HS256 and signing with the
                        server's public key to bypass asymmetric verification.

                

                

                    
🟡 Header Parameter kid

                    
Injecting a path traversal sequence into the kid parameter to point to a predictable
                        file (like /dev/null) and sign with an empty secret.

                

                

                    
🟡 JWK Header Injection

                    
Embedding an attacker-controlled public key directly in the jwk header and signing
                        the token with the corresponding private key.
