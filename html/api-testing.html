API Testing Research Index

            

                

                    
🔴 API Node Brute-force

                    
Submitting multiple variations of endpoint names (e.g., /api/v1/user,
                        /api/debug, /api/admin) to find hidden or undocumented functionality.
                    

                

                

                    
🔴 Mass Assignment

                    
Testing if injecting additional properties into a JSON update (e.g., adding
                        "role": "admin" to a profile update) allows for unauthorized privilege changes.

                

                

                    
🟡 BOLA / IDOR

                    
Testing for Broken Object Level Authorization by changing the ID in an API request (e.g.,
                        GET /user/1001 to GET /user/1002) to access other users' data.

                

                

                    
🟡 Versioning Bypass

                    
Testing if security controls applied to the latest API version (v2) are missing in legacy
                        versions (v1) that the server still supports and routes internally.
