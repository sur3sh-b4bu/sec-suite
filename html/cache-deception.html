Web Cache Deception Research Index

            

                

                    
🔴 Static Extension Abuse

                    
Testing if appending a static file extension (e.g., .js, .css) to a
                        sensitive endpoint (e.g., /settings) tricks a CDN into caching the private
                        response.

                

                

                    
🔴 Delimiter State Confusion

                    
Submitting URLs with non-standard delimiters like ; or ? (e.g.,
                        /profile;x.png) to exploit differences in how the cache and the origin server parse
                        the path.

                

                

                    
🟡 Path Translation Failures

                    
Exploiting normalization discrepancies where a CDN ignores encoded traversals (e.g.,
                        ..%2f) while the backend server resolves them to a sensitive directory.

                

                

                    
🟡 Cache Metadata Extraction

                    
Analyzing response headers like X-Cache, Age, and
                        Server-Timing to verify if an attacker-crafted URL has successfully populated the
                        edge cache nodes.
